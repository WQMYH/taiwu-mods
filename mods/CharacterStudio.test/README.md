# CharacterStudio.test

人物创建、谷密同步、批次关系与轮回信息注入 MOD 测试版。

## 当前功能

- 太吾完全保留玩家开局自定义结果。
- 可选解除开局属性、资质和特性点数限制。
- 谷密同步太吾六维、技艺资质、武学资质，并安全合并特性。
- 使用可自定义快捷键打开人物工坊，批量创建村民并在游戏内编辑模板。
- 五个分页集中管理村民、模板、谷密、传剑与轮回。
- 谷密支持姓名池、批次关系和人物模板。
- 太吾、谷密和村民均可选择复制自己、剑冢先人或随机模板作为前世。
- 详细设置保存到 `UserData/studio_settings.json`。
- `Languages/zh-Hans.lng` 与 `en-US.lng` 提供中英文界面。
- 初始村民、原版招募、新加入村民及现有村民可分别路由到不同模板。
- 过月只处理未应用模板或模板内容已经变化的人物。
- 数值 setter 写入后立即验证，失败时使用单项差值 API，再失败则回滚。
- 可选使用独立快捷键进入原版传剑流程。
- 可配置继承外观、姓名、六维、资质、立场和特性。
- 可选将前任太吾记录为前世，并向认识前任太吾的人物袒露身份、迁移关系。

## 传剑与转世

- 所有改变存档的传剑扩展默认关闭。
- 即刻传剑快捷键默认为 `T`；按下后仍需二次确认。
- “强制补满相枢感染”会永久改变当前太吾状态，启用前请备份存档。
- 开启袒露身份后，人物交互窗口会注入“袒露身份”选项；F9 面板仍保留按 NPC ID 调用的诊断入口。
- F9 面板提供“立即传剑”按钮，在快捷键冲突时可使用同一套确认和后端校验流程。
- 前世记录和关系迁移会写入存档，建议先在测试存档验证。

## 模板

预设与用户模板分开保存：

- `Profiles/preset_profiles.json`：随 MOD 提供的只读预设。
- `UserData/character_profiles.json`：游戏内保存的用户模板。
- `character_rules.json`：人物来源到模板 ID 的路由。
- `UserData/applied_state.json`：运行时生成的幂等状态。

内置 `vanilla_safe` 和 `full_positive_villager`。模板格式见
`Profiles/README.md`。

## 安全边界

- 不后置覆盖太吾。
- 不全局阻止住宅分配。
- 不替换原版家庭生成过程。
- 不使用固定特性 ID 取模判定正面特性。
- 轮回与关系数据只在对应设置开启并由玩家主动触发时处理。

## 测试顺序

1. 新建太吾，确认玩家自定义值未改变。
2. 检查谷密与太吾六维和两类资质一致。
3. F9 创建一个 `full_positive_villager`。
4. 检查日志中的 `before/target/after-setter`；如有回退，继续检查 `after-delta`。
5. 过月两次，确认第二次没有重复应用相同模板哈希。
6. 在备份存档中逐项开启传剑继承设置，确认关闭时保持原版结果。
7. 测试普通与强制即刻传剑、前世容量以及身份袒露的成功和拒绝路径。
