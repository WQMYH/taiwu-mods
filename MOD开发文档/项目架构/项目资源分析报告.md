### 项目信息

- **状态**: ✅ 活跃更新
- **技术栈**: .NET 8.0, HarmonyLib, SDK格式.csproj

### 项目结构
```
TaiwuMod_atakhalo/
├── TaiwuMod2025.sln                    # 主解决方案文件
├── InGameHelperBackend/                # 运行时助手后端（核心参考）
│   ├── InGameHelperBackend.cs          # 44KB代码
│   └── InGameHelperBackend.csproj      # 标准配置模板
├── InGameHelperFrontend/               # 前端UI
├── FeastExBackend/                     # 宴会增强后端
├── MinutiaeBackend/                    # 细节增强后端
└── ... (多个功能模块)
```

### 关键发现

#### ✅ 正确的.csproj配置模板
```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <PlatformTarget>x64</PlatformTarget>           <!-- 重要：指定x64 -->
    <TargetFramework>net8.0</TargetFramework>
    <ImplicitUsings>enable</ImplicitUsings>
    <Nullable>enable</Nullable>
    <OutputPath>...\Mod\运行时助手\Plugins\BackLib</OutputPath>
    <CopyLocalLockFileAssemblies>true</CopyLocalLockFileAssemblies>  <!-- 重要 -->
    <SatelliteResourceLanguages>en</SatelliteResourceLanguages>
  </PropertyGroup>
  
  <ItemGroup>
    <Reference Include="GameData">
      <HintPath>...\The Scroll Of Taiwu\Backend\GameData.dll</HintPath>
      <Private>False</Private>                      <!-- 重要：不复制DLL -->
    </Reference>
    <!-- 其他引用... -->
  </ItemGroup>
</Project>
```

#### 🔍 DLL引用路径规范
所有游戏DLL应该从以下位置引用：
- **Backend目录**: `...\The Scroll Of Taiwu\Backend\*.dll`
  - GameData.dll
  - GameData.Shared.dll
  - GameData.Serializer.dll
  - NLog.dll
  - Redzen.dll
  - Newtonsoft.Json.dll
  - TaiwuModdingLib.dll
  
- **Managed目录**: `...\The Scroll of Taiwu_Data\Managed\*.dll`
  - 0Harmony.dll
  - TaiwuModdingLib.dll (也可能在这里)

### 对我们的价值
1. ✅ **提供了生产级的.csproj配置模板**
2. ✅ **展示了正确的DLL引用方式**
3. ✅ **多模块架构设计值得学习**
4. ⚠️ **未找到SetBaseMainAttributes等API的使用示例**

### 关键代码模式

#### Harmony补丁注册
```csharp
public override void Initialize()
{
    harmony = new Harmony("Taiwu_CricketHacker.CricketHacker");
    
    // Postfix补丁
    PatchPostfix(
        AccessTools.Method(typeof(ViewCatchCricket), "InitCatchPlace"),
        nameof(InitCatchPlace_PostPatch));
    
    // Prefix补丁
    PatchPrefix(
        AccessTools.Method(typeof(ViewCatchCricket), "OnClickCatchPlace", 
                          new Type[] { typeof(int) }),
        nameof(OnClickCatchPlace_PrePatch));
}
```

### 对我们的价值
1. ✅ **展示了简单的Harmony Patch用法**

3. ️ **主要是UI层面的操作，与我们的Backend-only需求不完全匹配**
4. ⚠️ **未涉及Character属性设置等核心API**

### 项目信息
- **URL**: https://github.com/vizv/TaiwuDecompiler.git
- **状态**: 工具项目
- **功能**: 自动化反编译太吾绘卷游戏代码

### 项目结构
```
TaiwuDecompiler/
├── TaiwuDecompiler.sln
├── README.md                           # 使用说明
├── appveyor.yml                        # CI配置
├── ILSpy/                              # Git子模块（需初始化）
└── TaiwuDecompiler/
    ├── Program.cs                      # 194行核心逻辑
    ├── TaiwuDecompiler.csproj
    └── Utilities/
        ├── AssemblyUtility.cs          # Assembly加载和脱壳
        ├── HashUtility.cs              # SHA256计算
        └── RegistryUtility.cs          # 注册表读取
```

### 核心工作原理

#### 1. 自动获取游戏路径
```csharp
private static (string, string, string) GetSteamGameInstallation(uint gameId = 0)
{
    // 通过Steam注册表获取安装路径
    string steamGameRegistryKey = @"SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\Steam App " + gameId;
    string installLocationFromRegistry = RegistryUtility.LocalMachine.GetString(...);
    
    // 尝试多个候选路径
    List<string> candidates = new List<string> {
        installLocationFromRegistry,
        @"C:\Program Files (x86)\Steam\steamapps\common\The Scroll Of Taiwu",
        currentLocation,
    };
    
    foreach (string candidate in candidates) {
        if (File.Exists(GetAssemblyPath(candidate)) && 
            File.Exists(GetMonoPath(candidate))) {
            return (candidate, assemblyPath, monoPath);
        }
    }
}
```

#### 2. Assembly脱壳
```csharp
// 使用mono-2.0-bdwgc.dll加载并脱壳Assembly-CSharp.dll
byte[] unpackedAssemblyData = AssemblyUtility.LoadImage(monoLibraryPath, originalAssemblyPath);
File.WriteAllBytes(unpackedAssemblyPath, unpackedAssemblyData);
```

#### 3. ILSpy反编译
```csharp
WholeProjectDecompiler decompiler = new WholeProjectDecompiler();
var module = new PEFile(assemblyPath);
decompiler.AssemblyResolver = new UniversalAssemblyResolver(...);
decompiler.DecompileProject(module, outputPath);
```

### 可行性评估

#### ✅ 优点
1. **比dnSpy更专业** - 使用ILSpy的WholeProjectDecompiler
2. **自动化程度高** - 一键完成脱壳+反编译+清理
3. **输出结构清晰** - 生成完整的C#项目结构
4. **可以运行** - 只需要将工具复制到游戏目录即可

#### ⚠️ 缺点
1. **需要编译** - 需要先编译ILSpy子模块
2. **SSL证书问题** - git clone时遇到SSL验证失败
3. **法律风险** - README明确警告"切勿散播反编译后的源码"

#### 🎯 推荐用途
- ✅ **仅用于本地开发参考API**
- ✅ **查看Character类的完整方法列表**
- ✅ **查找SetBaseMainAttributes、GetAttraction等方法的正确签名**
-  **不要传播反编译结果**
-  **不要用于商业用途**

### 使用建议
1. 将`TaiwuDecompiler`目录复制到游戏根目录
2. 双击运行`TaiwuDecompiler.exe`（需要先编译）
3. 反编译结果会保存到当前目录的`TaiwuDecompiler-*`文件夹
4. **仅用于查看API，不要分享结果**
